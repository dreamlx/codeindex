# Release Notes v0.18.0 — Enriched Overview/Navigation README

**Date**: 2026-02-18

## Highlights

Overview and navigation level README_AI.md files (generated by `--fallback` mode) are now significantly more informative for AI agents.

### Before (v0.17.3)

```markdown
## Overview
- Modules: 1
- Files: 0
- Symbols: 0

## Modules
- **Application** - Module directory
```

### After (v0.18.0)

```markdown
## Overview
- Modules: 1
- Files: 5,128
- Symbols: 47,236

## Module Structure
project/
└── Application/
    ├── Admin/
    ├── Cashier/
    ├── Token/
    └── ... +33 more

## Modules
- **Application** - 5125 files | 47236 symbols

## Key Components
| Symbol | Type | Module |
|--------|------|--------|
| Config | class | Business |
| Token  | class | Token |
```

## Changes

### Added

- **Recursive statistics aggregation**: `_collect_recursive_stats()` aggregates file/symbol counts from child README_AI.md files. Overview and navigation levels now show accurate totals instead of zeros.
- **Smart module description**: `_extract_module_description()` rewritten with 3-strategy extraction (structured stats, class names, free-text fallback). Replaces generic "Module directory" with informative descriptions.
- **Key Components table**: Overview level surfaces top 15 class/function symbols across all modules with module attribution.
- **2-level directory tree**: Module Structure tree expands one level deeper, showing subdirectories of each top-level module (capped at 15 per module).
- **Real project validation framework**: `scripts/validate_real_projects.py` with 3-layer validation (L1 functional, L2 quality+AI, L3 experience), baseline comparison, and `make validate-*` targets.

## Validation Results

Tested against 3 real projects (php_admin, zcyl_backend, loomgraph):

| Metric | v0.17.3 | v0.18.0 |
|--------|:-------:|:-------:|
| architecture_comprehension | 1-2/10 | **3/10** |
| documentation_accuracy | 1-2/10 | **2-3/10** |
| symbol_navigation | 4-7/10 | 4-6/10 |

Remaining gaps (module purpose inference, dependency relationships) are semantic-layer concerns to be addressed by LoomGraph integration.
